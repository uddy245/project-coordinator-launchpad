# Narrated-slides lesson video pipeline

No-presenter lesson videos: on-brand slides rendered from the lesson script,
narrated by an ElevenLabs voice, stitched into an MP4 where each slide is held
for exactly the length of its narration. Scales to all 46 lessons — the
pipeline code never changes between lessons, only the per-lesson `segments.py`.

## Layout

```
scripts/video/
  build_common.py     Shared ffmpeg/timing helpers + lesson path resolution
  render_slides.py    segments.py -> lessons/<slug>/slides/NN.png  (Pillow)
  generate_audio.py   ← spends ElevenLabs credits: narration -> audio/NN.mp3
  build_video.py      slides + audio -> <slug>.mp4 (+ captions.srt + captions.vtt)
  make_preview.py     Silent estimated-timing preview (review aid, no credits)
  make_vtt.py         Backfill captions.vtt from existing captions.srt files
  lessons/
    lesson-20-raid-logs/
      segments.py     SINGLE SOURCE OF TRUTH: ordered slides + verbatim narration
      segments.json   Machine-readable export (validated by tests/unit/video-segments)
      slides/         Rendered PNGs (committed; small)
      narration/      Per-slide narration text + full_script.txt
      captions.srt    Time-aligned captions (regenerated by build_video.py)
```

`audio/`, the final `<slug>.mp4`, and `.work/` are gitignored — regenerable, and
the published MP4 lives in the Supabase `lesson-videos` bucket. The credit-costly
MP3s stay in `../video-production/<slug>/` as a local backup.

## The 3 steps

Each script takes an optional lesson slug (default `lesson-20-raid-logs`):

```bash
# 1. Slides (no network). Re-run after editing segments.py.
python3 scripts/video/render_slides.py lesson-20-raid-logs

# 2. Voiceover — YOUR ElevenLabs account, spends credits.
export ELEVENLABS_API_KEY="sk_..."        # elevenlabs.io > Profile (never commit)
export ELEVENLABS_VOICE_ID="..."          # an instant voice clone (warm senior-PM tone)
export ELEVENLABS_MODEL="eleven_multilingual_v2"
python3 scripts/video/generate_audio.py lesson-20-raid-logs
#   Re-record one slide: delete its audio/NN.mp3, then SKIP_EXISTING=1 python3 ...

# 3. Final build. Validates every clip with ffprobe and aborts on truncation.
python3 scripts/video/build_video.py lesson-20-raid-logs
#   -> lessons/lesson-20-raid-logs/lesson-20-raid-logs.mp4 (+ captions.srt)
```

Want to judge look/pacing before spending credits? `python3 scripts/video/make_preview.py`.

## Publishing into the app

The native `<video>` path (`src/components/lessons/video-player.tsx`) gives 10s
progress telemetry and the 90%-watched gate for free. Upload + set `video_url`:

```bash
# Dry run (default): prints the current vs. target video_url, uploads nothing.
node --env-file=.env.local scripts/seed-production-video.mjs lesson-20-raid-logs
# Apply: uploads the MP4 to the lesson-videos bucket and sets video_url + is_published.
node --env-file=.env.local scripts/seed-production-video.mjs lesson-20-raid-logs --apply
```

### Captions

`build_video.py` now also writes `captions.vtt` (sentence-level) alongside `captions.srt`
(slide-level). For a backfill of existing lessons:

```bash
# Regenerate captions.vtt from existing captions.srt — all lessons:
python3 scripts/video/make_vtt.py
# Single lesson:
python3 scripts/video/make_vtt.py lesson-20-raid-logs

# Upload all 25 VTT files to the lesson-videos bucket (dry run first):
node --env-file=.env.local scripts/upload-captions.mjs
node --env-file=.env.local scripts/upload-captions.mjs --apply
```

## Reusing for a new lesson

1. Write `docs/scripts/lesson-NN-<slug>.md`.
2. `cp -r lessons/lesson-20-raid-logs lessons/lesson-NN-<slug>` and replace
   `SEGMENTS` (verbatim narration + slide specs) and `FOOTER` in `segments.py`.
3. `render_slides.py` → `generate_audio.py` → `build_video.py` → seed. The
   pipeline code does not change.

## Requirements

`python3`, `Pillow`, `ffmpeg`/`ffprobe`, and `requests` (step 2 only).
```bash
pip install Pillow requests     # ffmpeg via your OS package manager (brew install ffmpeg)
```
