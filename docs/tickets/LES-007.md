# LES-007: Quiz player component

**Milestone:** M3 Lesson Delivery
**Dependencies:** LES-006, LES-008
**Estimated session length:** medium (30–90 min)
**Status:** not started

## Context

Client component: renders one question at a time, tracks answers in local state, submits via `submitQuizAttempt` action. On submit, shows score and per-item correct/incorrect + distractor rationale. 8/10 passes but failing does not block progression in MVP — we surface the score and offer "try again".

## Files to create

- `src/components/lessons/quiz-player.tsx`
- `src/actions/quiz.ts` — `submitQuizAttempt(lessonId, answers[])` server action; calls `gradeQuizAttempt` (from LES-008) and writes a `quiz_attempts` row

## Acceptance criteria

- [ ] All 10 items render one at a time with "Back / Next / Submit" navigation
- [ ] Progress indicator ("3 of 10") visible
- [ ] On submit, returns score and a results screen showing each item correct/incorrect + distractor rationale for the wrong answer chosen
- [ ] Attempt persists to `quiz_attempts` with user_id, lesson_id, score, answered_at, raw_answers jsonb
- [ ] Pass threshold 8/10 — failing shows "try again" button (does not clear progress for MVP)
- [ ] Server action returns `{ ok: true; data: { score, passed, perItem } } | { ok: false; error; code }`
