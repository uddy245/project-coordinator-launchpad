# LES-009: Per-lesson progress aggregation

**Milestone:** M3 Lesson Delivery
**Dependencies:** LES-004, LES-007
**Estimated session length:** short (≤30 min)
**Status:** not started

## Context

The `lesson_progress` table created in LES-004 has just the video columns. Extend it with `quiz_passed`, `artifact_submitted`, and a computed `status` (`not_started | in_progress | completed`). The dashboard card (LES-002) reads from this to render state. Status flipping to `completed` requires: video_watched=true AND quiz_passed=true AND artifact_submitted=true (the last flips in GRADE-003).

## Files to create / modify

- `supabase/migrations/20260109_lesson_progress_extend.sql` — add `quiz_passed boolean default false`, `artifact_submitted boolean default false` columns
- `src/lib/lessons/progress.ts` — `computeLessonStatus(progressRow): 'not_started' | 'in_progress' | 'completed'`
- Trigger in LES-007 quiz action to set `quiz_passed=true` on passing attempt (idempotent — ok to set twice)

## Acceptance criteria

- [ ] Passing the quiz (8/10 or better) flips `lesson_progress.quiz_passed=true`
- [ ] Watching ≥90% flips `lesson_progress.video_watched=true` (already done in LES-004)
- [ ] `computeLessonStatus` returns `not_started` when all flags false, `in_progress` when any flag true but not all, `completed` when all three true
- [ ] Dashboard lesson card reflects the computed status
