# FND-007: Vercel deployment

**Milestone:** M1 Foundation
**Dependencies:** FND-006
**Estimated session length:** short (≤30 min)
**Status:** not started

## Context

Connect the GitHub repo to a Vercel project. Preview deploys on every PR. Production tracks `main`. All env vars from `.env.example` must be set in Vercel Production and Preview environments.

This ticket requires access to Vercel dashboard — Claude Code can guide the human through it but cannot click buttons in Vercel.

## Acceptance criteria

- [ ] Pushing to `main` deploys to production on Vercel
- [ ] Opening a PR creates a preview deploy; Vercel comments the unique URL on the PR
- [ ] All env vars from `.env.example` are set in Vercel Production env
- [ ] Preview env has its own Supabase project (`launchpad-preview`) so it doesn't pollute prod data
- [ ] The landing page loads on production URL

## Tests required

None new.

## Definition of done

- [ ] Acceptance criteria checked
- [ ] Production URL noted in README
- [ ] PR opened (may be an empty commit PR marking this as done, or a README update PR)
