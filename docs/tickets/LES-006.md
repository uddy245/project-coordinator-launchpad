# LES-006: Quiz schema + seed 10 Lesson 20 items

**Milestone:** M3 Lesson Delivery
**Dependencies:** LES-001
**Estimated session length:** medium (30–90 min)
**Status:** not started

## Context

Multiple-choice quiz storage with one crucial constraint: the correct-answer column must NEVER be exposed via the anon key, so learners can't inspect answers from the client. Solution: a `public.quiz_items` table with full data, RLS-restricted so only `service_role` reads `correct`; plus a `public.quiz_items_public` view that projects everything except `correct`. Client-facing code reads the view; grading action uses the service role.

- `supabase/migrations/20260108_seed_quiz_lesson_20.sql` — 10 items

## Quiz item shape

```sql
create table public.quiz_items (
  id uuid primary key default gen_random_uuid(),
  lesson_id uuid not null references public.lessons(id) on delete cascade,
  sort integer not null,
  stem text not null,
  options jsonb not null,              -- [{id:'a', text:'...'}, ...]
  correct text not null,               -- 'a'|'b'|'c'|'d'
  distractor_rationale jsonb not null, -- { a:'...', b:'...' }
  competency text not null,
  difficulty text not null check (difficulty in ('easy','medium','hard')),
  created_at timestamptz not null default now()
);
```
