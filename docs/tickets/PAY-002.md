# PAY-002: Stripe webhook handler with signature verification

**Milestone:** M2 Auth & Checkout
**Dependencies:** PAY-001
**Estimated session length:** long (90 min+, plan to split if needed)
**Status:** not started

## Context

The webhook is where access is actually granted. Signature verified with `STRIPE_WEBHOOK_SECRET`. Idempotent on `stripe_session_id`. Writes to `purchases`, flips `profiles.has_access=true`.

This ticket is high-risk: a bug here can either (a) hand out free access or (b) fail to grant paid access. Both tests below are non-negotiable before merge.

## Questions to resolve before starting

- [ ] Webhook runs on Node runtime (not edge) — Stripe's Node SDK uses `stripe.webhooks.constructEvent` which needs Node crypto. Confirm in `route.ts`.
- [ ] `STRIPE_WEBHOOK_SECRET` differs between local (`stripe listen`), Preview, and Production. All three must be set.

## Files to create

- `src/app/api/stripe/webhook/route.ts` — POST handler, verifies signature, dispatches to handlers
- `src/lib/stripe/webhook.ts` — event handlers (`checkout.session.completed`, etc.) using the service-role Supabase client
- `supabase/migrations/20260103_purchases.sql` — `purchases` table with RLS and unique constraint on `stripe_session_id`
- `scripts/stripe-test-webhook.sh` — helper for local testing via Stripe CLI
- `tests/integration/stripe-webhook.test.ts`

## Acceptance criteria

- [ ] Webhook verifies Stripe signature; rejects with 400 if invalid (no DB writes)
- [ ] `checkout.session.completed` event inserts into `purchases` and updates `profiles.has_access=true`
- [ ] Duplicate event (same `stripe_session_id`) is a no-op — returns 200, inserts nothing (unique constraint enforces this)
- [ ] Failed/canceled payment events are logged but do not grant access
- [ ] `/checkout/success` page polls for `has_access=true` before transitioning to dashboard (max 10s, then show "still processing — refresh in a minute")
- [ ] Webhook route bypasses Next.js body parsing so the raw body is available for signature verification
