**Milestone:** M3 Lesson Delivery

## Context

Four gates per user (Foundation / Portfolio / Interview / Industry). MVP only computes a partial view of Gate 2 (Portfolio — "N of 7 artifacts completed"). Gates 1, 3, 4 remain `false` in MVP but the row and the compute function are structured so future work fills them in without schema changes.

## Files to create

- `supabase/migrations/20260110_gate_status.sql` — `gate_status` table with `user_id` PK, four booleans, `portfolio_artifacts_count int default 0`, RLS for own row
- `src/lib/gates/compute.ts` — pure function: `computeGateStatus(userProgress): GateStatus`
- Seed trigger: when `has_access` flips to true, insert a `gate_status` row with defaults
