## Acceptance criteria

- [ ] Returns correct score for all-correct, all-wrong, and mixed cases
- [ ] Handles unanswered items (not in `answers`) as incorrect, not as a crash
- [ ] Handles an answer referencing a non-existent item (ignores or returns an INVALID_INPUT-shaped error — pick one and test)
- [ ] Handles an answer with a non-existent choice (treated as wrong)
- [ ] Pass threshold is configurable, defaults to 8/10 but accepts an explicit override
- [ ] Pure — no DB, no I/O, no randomness

## Tests required

- [ ] Unit: ≥ 10 cases covering happy path, edge cases, and malformed input
