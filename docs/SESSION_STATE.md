## Local workflow (what to do EVERY ticket)

```bash
# 1. branch per ticket
git checkout main && git pull && git checkout -b tkt/<ID>-<slug>

# 2. edit code + tests

# 3. verify locally (env vars from .env.local)
pnpm typecheck
pnpm lint
set -a; source .env.local; set +a; pnpm test

# 4. for DB changes
supabase db push   # applies to launchpad-prod

# 5. for upload/grading/frontend changes — repro locally before pushing
node scripts/repro-upload.mjs   # Playwright-driven, exits with diagnostics

# 6. commit, push, open PR
git push -u origin <branch>
gh pr create --title "..." --body "..."

# 7. wait for CI, merge squash, delete branch
gh pr merge <#> --squash --delete-branch
```

### Non-negotiable rules (from CLAUDE.md, re-emphasized)

- **Reproduce locally before pushing** (my mistake earlier today; see
  `scripts/repro-upload.mjs` as the exemplar for new similar bugs)
- **`"use server"` files export async functions only** — no const/number/
  object exports (this bit us; move constants to a separate module)
- **Never edit an applied migration.** New migrations only.
- **Admin RLS policies using `is_admin()`** (SECURITY DEFINER) — never
  self-reference the target table inside the USING clause (recursion)
- **`supabase.auth.getUser()`, never `getSession()`** on the server
- **Server actions return `{ ok; data } | { ok: false; error; code }`** — never throw to the client
- **`revalidatePath`** in actions that change what a server component shows
  (we got stale dashboard cards without this)
